# [DEV-011] MiMeet 金流與發票整合規格書

**文檔版本：** v1.0  
**建立日期：** 2026年4月（Sprint 12）  
**適用範圍：** 綠界金流 + B2C 電子發票

---

## 1. 架構總覽

完全不依賴第三方金流套件，依據綠界官方技術文件直接串接：
- **金流 API：** 全方位金流 AioCheckOut/V5（[官方文件](https://developers.ecpay.com.tw/2509/)）
- **發票 API：** B2C 電子發票 Issue（[官方文件](https://developers.ecpay.com.tw/7809/)）

```
PaymentService::createOrder()
  → ECPayService::getPaymentUrl()
      ├─ sandbox → /mock（開發用）
      └─ production → AioCheckOut 表單 → cache → /checkout/{token}

ECPay Callback:
  POST /payments/ecpay/notify
    → verifyCallback (CheckMacValue SHA256)
    → 儲存對帳欄位 (TradeNo, PaymentDate, PaymentType)
    → activateSubscription
    → issueInvoiceForOrder (B2C 發票 API)
```

---

## 2. CheckMacValue 算法（SHA256）

1. 排除 CheckMacValue 欄位，參數按 key A→Z 排序
2. 前置 `HashKey=...&`，後置 `&HashIV=...`
3. URL encode（RFC 1866）
4. .NET 相容轉換：`%2d→- %5f→_ %2e→. %21→! %2a→* %28→( %29→)`
5. 全部小寫
6. SHA256 雜湊
7. 全部大寫

---

## 3. 發票 AES-128-CBC 加密

- **加密：** JSON → urlencode → AES-128-CBC（PKCS7） → Base64
- **解密：** Base64 → AES-128-CBC 解密 → urldecode → JSON
- Key/IV：使用發票專用 HashKey/HashIV

## 4. 金鑰管理

優先順序：`system_settings` DB → `config/services.php` → `.env`

| 設定 | system_settings key |
|------|-------------------|
| 金流 MerchantID | `ecpay.payment.merchant_id` |
| 金流 HashKey | `ecpay.payment.hash_key` |
| 金流 HashIV | `ecpay.payment.hash_iv` |
| 發票 MerchantID | `ecpay.invoice.merchant_id` |
| 發票 HashKey | `ecpay.invoice.hash_key` |
| 發票 HashIV | `ecpay.invoice.hash_iv` |
| 環境模式 | `ecpay.mode`（sandbox/production） |
| 發票啟用 | `ecpay.invoice.enabled` |

後台 UI：`/admin/settings` → 金流與發票 Tab → 即時切換環境

## 5. 訂單對帳欄位

`orders` 資料表：

| 欄位 | 來源 | 寫入時機 |
|------|------|---------|
| `ecpay_trade_no` | ECPay `TradeNo` | 收到 notify |
| `ecpay_payment_date` | ECPay `PaymentDate` | 收到 notify |
| `ecpay_payment_type` | ECPay `PaymentType` | 收到 notify |
| `ecpay_payment_type_charge_fee` | ECPay `ChargeFee` | 收到 notify |
| `invoice_no` | 發票 API `InvoiceNo` | 付款成功後 |
| `invoice_date` | 發票 API `InvoiceDate` | 付款成功後 |
| `invoice_random_number` | 發票 API `RandomNumber` | 付款成功後 |
