## 1. API 安全

### Rate Limiting

| 端點類型 | 限制 | 設定 |
|---------|------|------|
| 登入（前台 + 後台） | 5 次/分鐘 | `throttle:5,1` |
| 註冊 / 驗證 Email | 10 次/分鐘 | `throttle:10,1` |
| SMS OTP 發送 / 驗證 | 5 次/分鐘 | `throttle:5,1` |
| 一般 API（用戶） | 60 次/分鐘 | `throttle:60,1` |
| Admin API | 120 次/分鐘（由 API group 控制） | `throttle:api` |

### Authentication

| 場景 | 機制 | Token 有效期 |
|------|------|-------------|
| 前台 SPA | Sanctum Cookie + CSRF | 24 小時 |
| 後台管理 | Bearer Token（EnsureAdminUser middleware） | 8 小時 |
| 登出 | `currentAccessToken()->delete()` | 立即失效 |

## 2. 資料安全

### 密碼政策
- 最少 8 字元
- 必須包含英文字母（`regex:/[a-zA-Z]/`）
- 必須包含數字（`regex:/[0-9]/`）
- 儲存方式：bcrypt（`hashed` cast）

### 敏感欄位

| 欄位 | 儲存方式 | API 輸出 |
|------|---------|---------|
| password | bcrypt | `$hidden`（永不輸出） |
| remember_token | 明文 | `$hidden`（永不輸出） |
| phone | AES-256-CBC（`encrypted` cast） | 遮罩顯示 |

### Mass Assignment 防護
- `User::$fillable` 僅含用戶可編輯欄位
- `credit_score`、`status`、`membership_level` 等管理欄位不在 `$fillable` 中
- 管理操作使用 `$user->forceFill([...])->save()`

## 4. Broadcast Channel 安全

| 頻道 | 驗證方式 |
|------|---------|
| `chat.{conversationId}` | 驗證用戶為對話參與者（`user_a_id` 或 `user_b_id`） |
| `user.{userId}` | 驗證用戶 ID 一致 |
| `presence.chat.{conversationId}` | 參與者驗證 + 僅回傳 `id` + `nickname` |

---

## 5. 檔案上傳安全

- MIME 驗證：`mimes:jpeg,png,gif,webp`
- Magic bytes 驗證：`finfo_file()` 二次確認真實 MIME
- 大小限制：5MB（`max:5120`）
- 隔離區：刪除的檔案移至 `quarantine/` 而非直接刪除

---

## 6. 操作日誌

| 記錄範圍 | 機制 |
|---------|------|
| Admin 所有 POST/PATCH/DELETE | `LogAdminOperation` middleware 自動記錄 |
| 用戶登入 / 修改資料 / 上傳照片 / 變更手機 | `UserActivityLogService` 主動記錄 |
| 保留期限 | `data_retention_days` 設定（預設 180 天） |

---

## 7. 定期審查

- 套件漏洞掃描：每月執行 `composer audit` / `npm audit`
- Token 輪換：每季審查
- 安全標頭：每季對照 OWASP 最佳實務
