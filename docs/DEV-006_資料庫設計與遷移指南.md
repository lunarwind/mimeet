## 3. 完整資料表 DDL

### 3.1 核心用戶系統

#### users（用戶主表）

```sql
CREATE TABLE users (
  id                   BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  uuid                 CHAR(36)       UNIQUE NOT NULL  COMMENT 'UUID（對外識別用）',
  email                VARCHAR(255)   UNIQUE NOT NULL,
  phone                VARCHAR(255)   NULL             COMMENT 'AES-256 加密儲存（IV 隨機，無法直接 SQL 比對）',
  phone_hash           VARCHAR(64)    NULL UNIQUE      COMMENT 'SHA-256 of E.164 normalized phone — 用於 uniqueness 檢查（PR-5）',
  password             VARCHAR(255)   NOT NULL,
  nickname             VARCHAR(100)   NOT NULL,
  gender               ENUM('male','female','system') NOT NULL
                       COMMENT 'system=官方系統帳號（廣播DM用）',
  birth_date           DATE           NULL             COMMENT 'gender=system 時為 NULL',
  avatar_url           VARCHAR(500)   NULL,

  membership_level     TINYINT        NOT NULL DEFAULT 0
                       COMMENT '0=一般, 1=驗證, 2=進階驗證, 3=付費',
  is_paid              TINYINT(1)     NOT NULL DEFAULT 0
                       COMMENT '付費狀態快取（訂閱有效期間=1）',
  status               ENUM('active','suspended','auto_suspended','pending_deletion','deleted') NOT NULL DEFAULT 'active'
                       COMMENT 'active=正常 / suspended=管理員手動停權 / auto_suspended=誠信分數歸零自動停權（CreditScoreObserver 觸發） / pending_deletion=刪除冷靜期 / deleted=已軟刪除',
  credit_score         SMALLINT       NOT NULL DEFAULT 60
                       COMMENT '誠信分數 0–100，初始值 60',

  bio                  TEXT           NULL     COMMENT '自我介紹',
  height               SMALLINT UNSIGNED NULL COMMENT '身高 (cm)',
  weight               SMALLINT UNSIGNED NULL COMMENT '體重 (kg)',
  location             VARCHAR(100)   NULL     COMMENT '地區（縣市）',
  occupation           VARCHAR(100)   NULL     COMMENT '職業',
  education            VARCHAR(50)    NULL     COMMENT '學歷（high_school/associate/bachelor/master/phd/other）',
  interests            JSON           NULL     COMMENT '興趣標籤',
  profile              JSON           NULL     COMMENT '其他個人資料（擴充用）',
  privacy_settings     JSON           NULL     COMMENT '隱身/已讀回執/訪客記錄等設定',
  preferences          JSON           NULL     COMMENT '偏好搜尋條件',

  email_verified_at         TIMESTAMP      NULL,
  phone_verified_at         TIMESTAMP      NULL,
  advanced_verified_at      TIMESTAMP      NULL,
  credit_card_verified_at   TIMESTAMP      NULL     COMMENT '男性信用卡進階驗證完成時間（2026-04-26）',
  last_active_at       TIMESTAMP      NULL,
  created_at           TIMESTAMP      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at           TIMESTAMP      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at           TIMESTAMP      NULL     COMMENT '軟刪除',
  delete_requested_at  TIMESTAMP      NULL     COMMENT 'GDPR 刪除申請時間；排程每日 03:00 檢查超過 7 天者執行匿名化',

  INDEX idx_status_active  (status, last_active_at),
  INDEX idx_gender_birth   (gender, birth_date),
  INDEX idx_credit_score   (credit_score),
  FULLTEXT idx_search_nick (nickname)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='用戶主表；id=1 固定為 MiMeet 官方系統帳號';
```

> **id=1 系統帳號：** 廣播 DM 模式由此帳號發出私訊，接收者不可回覆。Seed 時固定建立。

> **Sprint 11 變更：** `membership_level` 欄位從 `TINYINT` 改為 `DECIMAL(3,1)`，以支援 Lv1.5（女性驗證等級）。有效值：0.0 / 1.0 / 1.5 / 2.0 / 3.0。
> ```
> // 2026_04_09_110001_alter_users_membership_level_to_decimal.php
> Schema::table('users', function (Blueprint $table) {
>     $table->decimal('membership_level', 3, 1)->default(0.0)
>           ->comment('0.0=一般, 1.0=驗證, 1.5=女性驗證, 2.0=進階驗證, 3.0=付費')
>           ->change();
> });
> ```

> **Sprint 15 變更：** `phone` 欄位從 `VARCHAR(20)` 擴展為 `VARCHAR(255)`，因應 Laravel `encrypted` cast（AES-256-CBC）加密後字串長度遠超 20 字元。
> ```
> // 2026_04_16_000001_widen_phone_column_for_encryption.php
> Schema::table('users', function (Blueprint $table) {
>     $table->string('phone', 255)->nullable()->change();
> });
> ```

> **Sprint 8 補充 Migration：**
> ```
> // 2026_04_09_000001_add_delete_requested_at_to_users_table.php
> Schema::table('users', function (Blueprint $table) {
>     $table->timestamp('delete_requested_at')->nullable()->after('deleted_at')
>           ->comment('GDPR 刪除申請時間；排程每日 03:00 檢查超過 7 天者執行匿名化');
> });
> ```

> **privacy_settings 預設值**（存於 users 表 JSON 欄位）：
> ```json
> {
>   "show_online_status": true,
>   "allow_profile_visits": true,
>   "show_in_search": true,
>   "show_last_active": true,
>   "allow_stranger_message": true
> }
> ```

#### user_verifications（身份驗證記錄）

```sql
CREATE TABLE user_verifications (
    id                    BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    user_id               BIGINT UNSIGNED NOT NULL,
    type                  ENUM('email','phone','photo','credit_card') NOT NULL,
    status                ENUM('pending','approved','rejected','expired') NOT NULL DEFAULT 'pending',
    token                 VARCHAR(100) NULL COMMENT 'OTP / 驗證連結 Token',
    token_expires_at      TIMESTAMP NULL,
    random_code           VARCHAR(10)  NULL
                          COMMENT '照片驗證用隨機碼（6位數字），10分鐘有效',
    verification_file_url VARCHAR(500) NULL COMMENT '照片驗證上傳的照片 URL',
    reject_reason         VARCHAR(200) NULL,
    reviewed_by           BIGINT UNSIGNED NULL COMMENT '審核管理員 ID（照片驗證）',
    reviewed_at           TIMESTAMP NULL,
    verified_at           TIMESTAMP NULL,
    created_at            TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at            TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    INDEX idx_user_type   (user_id, type),
    INDEX idx_status_type (status, type, created_at),
    CONSTRAINT fk_uv_user FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='用戶身份驗證記錄';
```

> **Sprint 11 擴充（女性驗證 Lv1.5）：** 此表新增 `random_code` 用於驗證照片隨機碼、`photo_url` 儲存驗證照片、`status` 擴充 `pending_code` / `pending_review` / `expired` 狀態、`expires_at` 驗證碼過期時間。Migration 如下：
> ```
> // 2026_04_09_110003_alter_user_verifications_for_female_verify.php
> Schema::table('user_verifications', function (Blueprint $table) {
>     // status ENUM 擴充為 'pending_code','pending_review','approved','rejected','expired'
>     $table->string('photo_url', 500)->nullable()->after('verification_file_url')
>           ->comment('女性驗證照片 URL（Lv1.5）');
>     $table->timestamp('expires_at')->nullable()->after('token_expires_at')
>           ->comment('驗證碼過期時間');
> });
> ```

```sql
CREATE TABLE conversations (
```

#### messages（聊天訊息）

```sql
CREATE TABLE messages (
    id                     BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
    uuid                   CHAR(36)        NOT NULL UNIQUE DEFAULT (UUID()),
    conversation_id        BIGINT UNSIGNED NOT NULL,
    sender_id              BIGINT UNSIGNED NOT NULL,
    type                   ENUM('text','image','qr_invite') NOT NULL DEFAULT 'text',
    content                TEXT NULL         COMMENT '文字訊息內容',
    image_url              VARCHAR(500) NULL COMMENT '圖片 URL（type=image）',
    meta                   JSON NULL         COMMENT '額外資料（如 QR 邀請 invitation_id）',
    is_read                TINYINT(1) NOT NULL DEFAULT 0,
    read_at                TIMESTAMP NULL,
    is_recalled            TINYINT(1) NOT NULL DEFAULT 0 COMMENT '5 分鐘內可回收',
    recalled_at            TIMESTAMP NULL,
    is_deleted_by_sender   TINYINT(1) NOT NULL DEFAULT 0,
    is_deleted_by_receiver TINYINT(1) NOT NULL DEFAULT 0,
    sent_at                TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    INDEX idx_conversation (conversation_id, sent_at DESC),
    INDEX idx_sender       (sender_id),
    FULLTEXT INDEX ft_content (content) COMMENT '支援後台關鍵字搜尋',
    CONSTRAINT fk_msg_conversation FOREIGN KEY (conversation_id)
        REFERENCES conversations(id) ON DELETE CASCADE,
    CONSTRAINT fk_msg_sender FOREIGN KEY (sender_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='聊天訊息';
```

### 5.9 註冊禁止名單(`registration_blacklists`,PR-2 2026-05-07 新增)

**用途**:禁止特定 email / mobile 重新註冊。與 `user_blocks`(user 對 user 互動封鎖)為**不同概念**。

**Schema 重點**:
- `type` ENUM('email','mobile')
- `value_hash` VARCHAR(64):normalize 後 SHA-256(email 走 lowercase+trim;mobile 走 `User::computePhoneHash`)
- `value_masked` VARCHAR(64):顯示用,呼叫 `\App\Support\Mask::email/phone`
- `is_active` BOOLEAN:**不用 SoftDeletes**(D8 — 避免重蹈 PR-1 user soft delete + unique 衝突)
- `active_value_hash` VARCHAR(64) NULL:方案 C race protection,active 時 = `value_hash`,deactivated 時 = null;Model `saving` event 自動同步
- `expires_at` TIMESTAMP NULL:null = 永久;BlacklistService 在查詢時過濾,**不靠 cron 清**
- `source` ENUM('manual','admin_delete'):記錄來源
- `source_user_id`:若 source='admin_delete' 指向被刪 user 舊 id

**索引**:
- `UNIQUE(type, active_value_hash)` — schema-level race protection(MySQL UNIQUE 允許多 NULL → 可保留完整歷史)
- `INDEX(type, value_hash)` — 註冊查詢用
- `INDEX(expires_at)` / `INDEX(source_user_id)` / `INDEX(created_by)`

詳細 schema 與 migration 範例見 `2026_05_07_100626_create_registration_blacklists_table.php`。

### 6.1 上傳架構（後端中轉模式）

```
前台上傳流程：
  1. 前台 → POST /api/v1/uploads （multipart/form-data）
  2. 後端：驗證格式 / 大小 / MIME type
  3. 後端：呼叫 Google Vision SafeSearch AI 審核
  4. 通過 → 轉換 WebP → 上傳 DO Spaces → 回傳 CDN URL
```

不使用 Presigned URL 直傳，確保所有檔案通過後端審核後才儲存。

## 7. 索引設計原則

1. **主鍵**：所有表必須有 `id BIGINT UNSIGNED AUTO_INCREMENT`
2. **UUID**：對外暴露的識別符使用 UUID，避免洩漏資料量
3. **外鍵**：必須建立外鍵約束，ON DELETE CASCADE 或 SET NULL
4. **查詢索引**：WHERE 條件常用欄位必須加索引
5. **複合索引**：`ORDER BY + WHERE` 組合查詢建立複合索引（WHERE 欄位在前）
6. **時間索引**：含時間範圍查詢的表，時間欄位需加索引

### 7.1 禁止事項

```sql
-- ❌ 禁止在 Migration 中使用原生 SQL 字串拼接
DB::statement("ALTER TABLE users ADD INDEX ...");

-- ✅ 使用 Blueprint 方法
$table->index(['status', 'last_active_at']);
```
