## 3. 完整資料表 DDL

### 3.1 核心用戶系統

#### users（用戶主表）

```sql
CREATE TABLE users (
  id                   BIGINT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
  uuid                 CHAR(36)       UNIQUE NOT NULL  COMMENT 'UUID（對外識別用）',
  email                VARCHAR(255)   UNIQUE NOT NULL,
  phone                VARCHAR(255)   NULL             COMMENT 'AES-256 加密儲存',
  password             VARCHAR(255)   NOT NULL,
  nickname             VARCHAR(100)   NOT NULL,
  gender               ENUM('male','female','system') NOT NULL
                       COMMENT 'system=官方系統帳號（廣播DM用）',
  birth_date           DATE           NULL             COMMENT 'gender=system 時為 NULL',
  avatar_url           VARCHAR(500)   NULL,

  membership_level     TINYINT        NOT NULL DEFAULT 0
                       COMMENT '0=一般, 1=驗證, 2=進階驗證, 3=付費',
  is_paid              TINYINT(1)     NOT NULL DEFAULT 0
                       COMMENT '付費狀態快取（訂閱有效期間=1）',
  status               ENUM('active','suspended','pending_deletion','deleted') NOT NULL DEFAULT 'active',
  credit_score         SMALLINT       NOT NULL DEFAULT 60
                       COMMENT '誠信分數 0–100，初始值 60',

  profile              JSON           NULL     COMMENT '身高/體重/職業/地區/自我介紹等',
  privacy_settings     JSON           NULL     COMMENT '隱身/已讀回執/訪客記錄等設定',
  preferences          JSON           NULL     COMMENT '偏好搜尋條件',

  email_verified_at    TIMESTAMP      NULL,
  phone_verified_at    TIMESTAMP      NULL,
  advanced_verified_at TIMESTAMP      NULL,
  last_active_at       TIMESTAMP      NULL,
  created_at           TIMESTAMP      NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at           TIMESTAMP      NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at           TIMESTAMP      NULL     COMMENT '軟刪除',
  delete_requested_at  TIMESTAMP      NULL     COMMENT 'GDPR 刪除申請時間；排程每日 03:00 檢查超過 7 天者執行匿名化',

  INDEX idx_status_active  (status, last_active_at),
  INDEX idx_gender_birth   (gender, birth_date),
  INDEX idx_credit_score   (credit_score),
  FULLTEXT idx_search_nick (nickname)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci
  COMMENT='用戶主表；id=1 固定為 MiMeet 官方系統帳號';
```

> **id=1 系統帳號：** 廣播 DM 模式由此帳號發出私訊，接收者不可回覆。Seed 時固定建立。

> **Sprint 8 補充 Migration：**
> ```
> // 2026_04_09_000001_add_delete_requested_at_to_users_table.php
> Schema::table('users', function (Blueprint $table) {
>     $table->timestamp('delete_requested_at')->nullable()->after('deleted_at')
>           ->comment('GDPR 刪除申請時間；排程每日 03:00 檢查超過 7 天者執行匿名化');
> });
> ```

> **privacy_settings 預設值**（存於 users 表 JSON 欄位）：
> ```json
> {
>   "show_online_status": true,
>   "allow_profile_visits": true,
>   "show_in_search": true,
>   "show_last_active": true,
>   "allow_stranger_message": true
> }
> ```

## 6. 媒體上傳規格

### 6.1 上傳架構（後端中轉模式）

```
前台上傳流程：
  1. 前台 → POST /api/v1/uploads （multipart/form-data）
  2. 後端：驗證格式 / 大小 / MIME type
  3. 後端：呼叫 Google Vision SafeSearch AI 審核
  4. 通過 → 轉換 WebP → 上傳 DO Spaces → 回傳 CDN URL
  5. 未通過 → HTTP 451，error_code: 4510
```

不使用 Presigned URL 直傳，確保所有檔案通過後端審核後才儲存。

### 6.2 圖片規格限制

| 用途 | 最大大小 | 允許格式 | 尺寸處理 |

| 回報佐證圖片 | 5 MB | jpg, png, webp | 縮圖至長邊 1200px |

## 7. 索引設計原則

1. **主鍵**：所有表必須有 `id BIGINT UNSIGNED AUTO_INCREMENT`
2. **UUID**：對外暴露的識別符使用 UUID，避免洩漏資料量
3. **外鍵**：必須建立外鍵約束，ON DELETE CASCADE 或 SET NULL
4. **查詢索引**：WHERE 條件常用欄位必須加索引
5. **複合索引**：`ORDER BY + WHERE` 組合查詢建立複合索引（WHERE 欄位在前）
6. **時間索引**：含時間範圍查詢的表，時間欄位需加索引

### 7.1 禁止事項

```sql
-- ❌ 禁止在 Migration 中使用原生 SQL 字串拼接
DB::statement("ALTER TABLE users ADD INDEX ...");

-- ✅ 使用 Blueprint 方法
$table->index(['status', 'last_active_at']);
```
