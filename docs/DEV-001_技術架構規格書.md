## 1. 專案概述

MiMeet 是一個針對台灣市場的高端包養交友平台，核心特色為：
- **誠信分數系統**：動態評估用戶可信度，控制平台互動權限
- **QR碼約會驗證**：創新的線下見面驗證機制
- **多重身份驗證**：Email / 手機 / 進階（照片/信用卡）三層驗證
- **匿名聊天室**：第二階段功能，提供匿名公開互動空間

### 1.1 開發範疇

| 範疇 | 說明 |
|------|------|
| 前台 Web | Vue.js 3 SPA，面向一般用戶（甜爹/甜心/經紀公司） |
| 後台管理 | React 18 SPA，面向平台管理員（super_admin / admin / cs） |
| 後端 API | Laravel 10 RESTful API + WebSocket |
| 資料庫 | MySQL 8.0（主要資料）+ Redis 7（快取/會話/佇列） |
| 儲存 | DigitalOcean Spaces（圖片/文件），本地 CDN 加速 |

## 5. 資料流與通訊模式

### 5.1 REST API（主要通訊）

```
前台/後台 → HTTPS Request → Nginx → Laravel API
                                         ↓
                                  Middleware Stack
                                  (Auth, RBAC, RateLimit)
                                         ↓
                                  Controller → Service → Model
                                         ↓
                                  JSON Response
```

### 5.2 WebSocket（即時通訊）

```
前台 Socket.IO ─── connect ──→ Laravel Reverb
                               ↓
                         頻道訂閱（private/presence）
                               ↓
                    私訊頻道: private-chat.{user_id}
                    通知頻道: private-notify.{user_id}
                    聊天室頻道: presence-anon-channel.{channel_id}
```

### 5.3 非同步任務（Queue）

```
觸發事件 → Laravel Queue (Redis) → Horizon Worker
                                          ↓
                              ┌─────────────────────┐
                              │  Email 驗證通知      │
                              │  SMS 驗證碼          │
                              │  推送通知            │
                              │  圖片 AI 審核        │
                              │  Stats 快取更新      │
                              └─────────────────────┘
```

## 6. 安全架構

### 6.1 認證機制

| 場景 | 機制 |
|------|------|
| 前台 SPA 用戶 | Laravel Sanctum Cookie（SPA 模式）+ CSRF Token |
| 後台管理員 | Laravel Sanctum Token（Bearer Token） |
| WebSocket 連線 | Sanctum Token 認證 |
| 第三方 Webhook（綠界回呼） | HMAC 簽章驗證 |

### 6.2 RBAC 角色矩陣

| 功能模組 | super_admin | admin | cs（客服） |
|---------|:-----------:|:-----:|:---------:|
| 會員瀏覽 | ✅ | ✅ | ✅ |
| 會員操作（調分/停權） | ✅ | ✅ | ❌ |
| 會員刪除 | ✅ | ❌ | ❌ |
| Ticket 查看 | ✅ | ✅ | ✅ |
| Ticket 處理 | ✅ | ✅ | ✅ |
| 聊天記錄查詢 | ✅ | ✅ | ❌ |
| 支付記錄查詢 | ✅ | ✅ | ❌ |
| SEO 管理 | ✅ | ✅ | ❌ |
| 系統設定/角色管理 | ✅ | ❌ | ❌ |
| 訂閱方案/折扣設定 | ✅ | ✅ | ❌ |

### 6.3 資料安全

- **傳輸加密**：全站 HTTPS / TLS 1.3
- **密碼儲存**：bcrypt（cost factor 12）
- **敏感欄位加密**：手機號碼、信用卡末四碼使用 AES-256-CBC（Laravel Crypt）
- **JWT / Sanctum Token 有效期**：access token 24h，refresh token 30d
- **Rate Limiting**：登入 API 每 IP 每分鐘 5 次，一般 API 每用戶每分鐘 60 次
- **SQL 注入防護**：Eloquent ORM 參數綁定，禁止原始 SQL 字串拼接
- **XSS 防護**：Response 資料全面 htmlspecialchars，Blade 範本自動 escape
- **CSRF 防護**：前台 SPA 使用 Sanctum Cookie 模式，自動 CSRF 驗證

## 7. 性能目標與對應設計

| 指標 | 目標 | 實現手段 |
|------|------|----------|
| API 響應時間 | p95 < 300ms | Redis 快取熱點資料、DB 索引優化 |
| 頁面首次載入 | < 3 秒 | Vite 代碼分割、CDN、圖片 WebP 格式 |
| 聊天訊息延遲 | < 1 秒 | WebSocket（Laravel Reverb） |
| 同時在線用戶 | 2,500+ | Nginx 負載均衡、Redis Session |
| 系統可用性 | 99.9% | Docker 健康檢查、AWS 多 AZ 部署 |
