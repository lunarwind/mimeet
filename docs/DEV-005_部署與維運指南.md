### 1.1 建議 Droplet 規格

| 階段 | 規格 | 月費（約） | 適用時機 |
|------|------|-----------|----------|
| 初期上線 | 4 vCPU / 8 GB RAM / 160 GB SSD | ~$48 USD | 0–1,000 用戶 |
| 成長期 | 8 vCPU / 16 GB RAM / 320 GB SSD | ~$96 USD | 1,000–3,000 用戶 |
| 擴展期 | 獨立 DB Droplet + App Droplet | ~$144 USD | 3,000+ 用戶 |

> 初期以單一 Droplet 跑所有服務，待流量成長再拆分資料庫至獨立 Droplet。

## 2. Droplet 初始設定

### 2.1 建立 Droplet

1. 至 DigitalOcean 控制台建立 Droplet
   - Image：**Ubuntu 22.04 LTS x64**
   - Plan：依上表選擇（建議初期 4 vCPU / 8 GB）
   - 區域：SGP1（新加坡）或 NRT1（東京）
   - Authentication：**SSH Key**（禁止密碼登入）
   - Backups：**啟用**（每週自動備份）

2. 取得 Droplet IP，至 CloudFlare 設定 DNS：
   ```
   A  mimeet.tw       → {Droplet IP}   （Proxy: ON）
   A  www.mimeet.tw   → {Droplet IP}   （Proxy: ON）
   A  api.mimeet.tw   → {Droplet IP}   （Proxy: ON）
   ```

### 2.2 基礎安全設定

```bash
# 以 root 登入後執行
ssh root@{Droplet-IP}

# 建立部署帳號
adduser deploy
usermod -aG sudo deploy
usermod -aG docker deploy

# 複製 SSH 金鑰給 deploy 帳號
rsync --archive --chown=deploy:deploy ~/.ssh /home/deploy

# 防火牆（UFW）
ufw default deny incoming
ufw default allow outgoing
ufw allow ssh        # 22
ufw allow 80         # HTTP
ufw allow 443        # HTTPS
ufw enable

# 禁止 root 密碼登入
sed -i 's/#PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
sed -i 's/PermitRootLogin yes/PermitRootLogin prohibit-password/' /etc/ssh/sshd_config
systemctl reload sshd

# 系統更新
apt update && apt upgrade -y
apt install -y git curl wget unzip fail2ban
```

### 2.3 安裝 Docker

```bash
# 官方安裝腳本
curl -fsSL https://get.docker.com | bash
apt install -y docker-compose-plugin

# 開機自啟
systemctl enable docker && systemctl start docker

# 確認版本
docker --version
docker compose version
```

### 2.4 安裝 Nginx（Host 主機）

```bash
apt install -y nginx certbot python3-certbot-nginx
systemctl enable nginx
```

## 4. Docker Compose 生產設定

### 4.1 `/var/www/mimeet/docker-compose.prod.yml`

```yaml
version: '3.9'

services:
  app:
    image: ghcr.io/{your-org}/mimeet-backend:${IMAGE_TAG:-latest}
    container_name: mimeet_app
    restart: unless-stopped
    volumes:
      - /var/www/mimeet/backend/storage:/var/www/html/storage
      - /var/www/mimeet/backend/.env:/var/www/html/.env:ro
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    networks: [mimeet_net]

  reverb:
    image: ghcr.io/{your-org}/mimeet-backend:${IMAGE_TAG:-latest}
    container_name: mimeet_reverb
    restart: unless-stopped
    command: php artisan reverb:start --host=0.0.0.0 --port=8080
    volumes:
      - /var/www/mimeet/backend/.env:/var/www/html/.env:ro
    depends_on: [app]
    networks: [mimeet_net]

  horizon:
    image: ghcr.io/{your-org}/mimeet-backend:${IMAGE_TAG:-latest}
    container_name: mimeet_horizon
    restart: unless-stopped
    command: php artisan horizon
    volumes:
      - /var/www/mimeet/backend/.env:/var/www/html/.env:ro
    depends_on: [app]
    networks: [mimeet_net]

  scheduler:
    image: ghcr.io/{your-org}/mimeet-backend:${IMAGE_TAG:-latest}
    container_name: mimeet_scheduler
    restart: unless-stopped
    command: sh -c "while true; do php artisan schedule:run --verbose --no-interaction; sleep 60; done"
    volumes:
      - /var/www/mimeet/backend/.env:/var/www/html/.env:ro
    depends_on: [app]
    networks: [mimeet_net]

  mysql:
    image: mysql:8.0
    container_name: mimeet_mysql
    restart: unless-stopped
    env_file: /var/www/mimeet/backend/.env
    environment:
      MYSQL_DATABASE: ${DB_DATABASE}
      MYSQL_USER: ${DB_USERNAME}
      MYSQL_PASSWORD: ${DB_PASSWORD}
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
    volumes:
      - /var/lib/mimeet/mysql:/var/lib/mysql
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost", "-uroot", "-p${DB_ROOT_PASSWORD}"]
      interval: 10s
      timeout: 5s
      retries: 5
    networks: [mimeet_net]

  redis:
    image: redis:7.0-alpine
    container_name: mimeet_redis
    restart: unless-stopped
    command: redis-server --requirepass ${REDIS_PASSWORD} --save 60 1
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "--no-auth-warning", "-a", "${REDIS_PASSWORD}", "ping"]
      interval: 10s
      retries: 5
    networks: [mimeet_net]

volumes:
  redis_data:

networks:
  mimeet_net:
    driver: bridge
```

## 5. Nginx 設定

### 5.1 申請 SSL 憑證

```bash
# DNS 確認指向此 IP 後執行
certbot --nginx -d mimeet.tw -d www.mimeet.tw -d api.mimeet.tw
```

### 5.2 `/etc/nginx/sites-available/mimeet`

```nginx
# ── 前台（Vue.js SPA）──────────────────────────────
server {
    listen 443 ssl http2;
    server_name mimeet.tw www.mimeet.tw;

    ssl_certificate     /etc/letsencrypt/live/mimeet.tw/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/mimeet.tw/privkey.pem;
    include             /etc/letsencrypt/options-ssl-nginx.conf;

    root  /var/www/mimeet/frontend-dist;
    index index.html;

    # SPA fallback
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 靜態資源長快取
    location ~* \.(js|css|png|jpg|jpeg|webp|ico|svg|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
        access_log off;
    }

    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
}

# ── 後台（React SPA）───────────────────────────────
# 可與前台共用一個 server block，或拆成獨立子網域
# 此處以 /admin/ 路徑方式整合到前台 server block
# （在上面 server block 的 location 段加入以下）:
#   location /admin/ {
#     alias /var/www/mimeet/admin-dist/;
#     try_files $uri $uri/ /admin/index.html;
#   }

# ── Laravel API ─────────────────────────────────────
server {
    listen 443 ssl http2;
    server_name api.mimeet.tw;

    ssl_certificate     /etc/letsencrypt/live/mimeet.tw/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/mimeet.tw/privkey.pem;
    include             /etc/letsencrypt/options-ssl-nginx.conf;

    root  /var/www/mimeet/backend/public;
    index index.php;

    # API → PHP-FPM（Docker 容器，透過 Docker network）
    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass   mimeet_app:9000;
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include        fastcgi_params;
        fastcgi_read_timeout 300;
    }

    # WebSocket（Reverb）
    location /app/ {
        proxy_pass         http://mimeet_reverb:8080;
        proxy_http_version 1.1;
        proxy_set_header   Upgrade $http_upgrade;
        proxy_set_header   Connection "Upgrade";
        proxy_set_header   Host $host;
        proxy_read_timeout 3600s;
    }

    # Horizon 監控（限制 IP）
    location /horizon {
        allow {你的辦公室IP};
        deny  all;
        try_files $uri $uri/ /index.php?$query_string;
    }

    client_max_body_size 20M;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;
}

# ── HTTP → HTTPS 重導向 ─────────────────────────────
server {
    listen 80;
    server_name mimeet.tw www.mimeet.tw api.mimeet.tw;
    return 301 https://$host$request_uri;
}
```

```bash
# 啟用 vhost
ln -s /etc/nginx/sites-available/mimeet /etc/nginx/sites-enabled/
nginx -t && systemctl reload nginx
```

## 6. CI/CD 管道（GitHub Actions）

### 6.1 `.github/workflows/deploy.yml`

```yaml
name: Deploy to DigitalOcean

on:
  push:
    branches: [main]

env:
  REGISTRY: ghcr.io
  IMAGE_NAME: ${{ github.repository }}/mimeet-backend

jobs:
  # ── 後端測試 ─────────────────────────────────────
  test-backend:
    runs-on: ubuntu-latest
    services:
      mysql:
        image: mysql:8.0
        env:
          MYSQL_DATABASE: mimeet_test
          MYSQL_USER: mimeet
          MYSQL_PASSWORD: secret
          MYSQL_ROOT_PASSWORD: root
        options: --health-cmd="mysqladmin ping" --health-interval=10s
      redis:
        image: redis:7-alpine
        options: --health-cmd="redis-cli ping" --health-interval=10s
    steps:
      - uses: actions/checkout@v4
      - uses: shivammathur/setup-php@v2
        with: { php-version: '8.2', extensions: pdo_mysql, redis, gd, zip, opcache }
      - run: composer install --no-dev --optimize-autoloader
        working-directory: ./backend
      - run: cp .env.ci .env && php artisan key:generate && php artisan migrate --force
        working-directory: ./backend
      - run: php artisan test --coverage --min=80
        working-directory: ./backend

  # ── 建置後端 Docker Image ─────────────────────────
  build-backend:
    needs: test-backend
    runs-on: ubuntu-latest
    permissions: { contents: read, packages: write }
    outputs:
      short_sha: ${{ steps.sha.outputs.short }}
    steps:
      - uses: actions/checkout@v4
      - id: sha
        run: echo "short=$(echo $GITHUB_SHA | cut -c1-7)" >> $GITHUB_OUTPUT

      - uses: docker/login-action@v3
        with:
          registry: ${{ env.REGISTRY }}
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - uses: docker/build-push-action@v5
        with:
          context: ./backend
          push: true
          tags: |
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:${{ steps.sha.outputs.short }}
            ${{ env.REGISTRY }}/${{ env.IMAGE_NAME }}:latest
          cache-from: type=gha
          cache-to: type=gha,mode=max

  # ── 建置前台 / 後台 ───────────────────────────────
  build-frontend:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: 'npm', cache-dependency-path: frontend/package-lock.json }
      - run: npm ci && npm run type-check && npm run lint && npm run test && npm run build
        working-directory: ./frontend
        env:
          VITE_API_BASE_URL: https://api.mimeet.tw/api/v1
          VITE_WS_HOST: api.mimeet.tw
          VITE_WS_PORT: 443
          VITE_WS_SCHEME: wss
          VITE_WS_KEY: ${{ secrets.REVERB_APP_KEY }}
      - uses: actions/upload-artifact@v4
        with: { name: frontend-dist, path: frontend/dist/, retention-days: 1 }

  build-admin:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: 'npm', cache-dependency-path: admin/package-lock.json }
      - run: npm ci && npm run build
        working-directory: ./admin
        env:
          VITE_API_BASE_URL: https://api.mimeet.tw/api/v1
      - uses: actions/upload-artifact@v4
        with: { name: admin-dist, path: admin/dist/, retention-days: 1 }

  # ── 部署至 DigitalOcean Droplet ───────────────────
  deploy:
    needs: [build-backend, build-frontend, build-admin]
    runs-on: ubuntu-latest
    environment: production
    steps:
      - uses: actions/checkout@v4

      - uses: actions/download-artifact@v4
        with: { name: frontend-dist, path: ./frontend-dist }
      - uses: actions/download-artifact@v4
        with: { name: admin-dist, path: ./admin-dist }

      # 上傳靜態檔案到 Droplet
      - uses: appleboy/scp-action@v0.1.7
        with:
          host: ${{ secrets.DO_HOST }}
          username: deploy
          key: ${{ secrets.DO_SSH_KEY }}
          source: "frontend-dist/,admin-dist/"
          target: /tmp/mimeet-assets/

      # SSH 執行部署
      - uses: appleboy/ssh-action@v1.0.3
        with:
          host: ${{ secrets.DO_HOST }}
          username: deploy
          key: ${{ secrets.DO_SSH_KEY }}
          script: |
            set -e
            SHORT_SHA="${{ needs.build-backend.outputs.short_sha }}"
            COMPOSE="docker compose -f /var/www/mimeet/docker-compose.prod.yml"

            echo "=== [1/6] 更新靜態資源 ==="
            cp -r /var/www/mimeet/frontend-dist /var/www/mimeet/frontend-dist.bak 2>/dev/null || true
            rsync -a --delete /tmp/mimeet-assets/frontend-dist/ /var/www/mimeet/frontend-dist/
            rsync -a --delete /tmp/mimeet-assets/admin-dist/ /var/www/mimeet/admin-dist/

            echo "=== [2/6] 拉取最新 Docker Image ==="
            echo "${{ secrets.GHCR_TOKEN }}" | docker login ghcr.io -u "${{ github.actor }}" --password-stdin
            export IMAGE_TAG=$SHORT_SHA
            $COMPOSE pull app reverb horizon scheduler

            echo "=== [3/6] 執行 Migration ==="
            $COMPOSE run --rm app php artisan migrate --force

            echo "=== [4/6] 滾動重啟服務 ==="
            $COMPOSE up -d --no-deps --remove-orphans app reverb horizon scheduler

            echo "=== [5/6] 清除 Laravel 快取 ==="
            $COMPOSE exec -T app php artisan optimize

            echo "=== [6/6] 重載 Nginx ==="
            sudo nginx -t && sudo systemctl reload nginx

            echo "✅ 部署完成：$SHORT_SHA"
```

## 9. 資料庫備份

### 9.1 每日自動備份腳本

```bash
cat > /opt/scripts/backup-mysql.sh << 'EOF'
#!/bin/bash
set -e
BACKUP_DIR="/var/backups/mimeet-mysql"
DATE=$(date +%Y%m%d_%H%M%S)
RETENTION=14

mkdir -p "$BACKUP_DIR"

source /var/www/mimeet/backend/.env

docker exec mimeet_mysql \
  mysqldump -uroot -p"${DB_ROOT_PASSWORD}" \
  --single-transaction --routines --triggers \
  mimeet | gzip > "$BACKUP_DIR/mimeet_${DATE}.sql.gz"

find "$BACKUP_DIR" -name "*.sql.gz" -mtime +${RETENTION} -delete

echo "[$(date)] 備份完成：mimeet_${DATE}.sql.gz"
EOF

chmod +x /opt/scripts/backup-mysql.sh

# Cron：每天凌晨 3 點
echo "0 3 * * * deploy /opt/scripts/backup-mysql.sh >> /var/log/mimeet-backup.log 2>&1" | crontab -
```

### 9.2 同步至 DO Spaces（異地備份）

```bash
# 安裝 s3cmd
apt install -y s3cmd

# 設定（填入 Spaces 金鑰）
s3cmd --configure

# 在備份腳本末尾加入
s3cmd put "$BACKUP_DIR/mimeet_${DATE}.sql.gz" s3://mimeet-backups/mysql/
```

## 10. 監控設定

## 10.4 排程任務一覽（Scheduled Commands）

排程透過 `mimeet-scheduler` container 觸發（每分鐘執行 `php artisan schedule:run`）。
新增 / 修改排程必須同步更新本表。

| Command | 頻率 | 用途 | Log 位置 |
|---|---|---|---|
| `gdpr:process-deletions` | 每日 03:00 | 處理 GDPR 帳號刪除請求 | Laravel default |
| `payments:auto-refund-verifications` | 每日 03:00 | 派發到期信用卡驗證退款 Job | Laravel default |
| `payments:reconcile-ecpay` | 每 15 分鐘 | ECPay 對帳輪詢 | Laravel default |
| `subscriptions:expire` | 每日 00:05 | 過期訂閱：標記 expired + 降級 user.membership_level + 站內通知 | `storage/logs/subscriptions-expire.log` |
| `subscriptions:notify-expiring` | 每日 09:00 | 對 3 天內到期的訂閱發送站內提醒（每訂閱限通知一次） | `storage/logs/subscriptions-notify-expiring.log` |

排程設計原則：
- 所有排程必須加 `->withoutOverlapping()->onOneServer()`（避免多 server 重複跑）
- 重要排程加 `->appendOutputTo(storage_path('logs/...'))` 便於 debug
- 排程 command 內部處理失敗應 `try/catch + Log::error`，不讓單筆失敗中斷整批

確認排程已註冊：
```bash
docker exec mimeet-app php artisan schedule:list
```

確認 mimeet-scheduler 運作中：
```bash
docker compose -f docker-compose.staging.yml ps scheduler
docker compose -f docker-compose.staging.yml logs --tail=20 scheduler
```

## 11. 維運常用指令速查

```bash
# SSH 進 Droplet
ssh deploy@{Droplet-IP}

# 切換專案目錄
cd /var/www/mimeet
COMPOSE="docker compose -f docker-compose.prod.yml"

# 查看容器狀態
$COMPOSE ps

# 查看即時 Log
$COMPOSE logs -f app
$COMPOSE logs -f horizon

# 執行 Artisan 指令
$COMPOSE exec app php artisan {command}

# 清除 Laravel 快取
$COMPOSE exec app php artisan optimize:clear

# 進入 MySQL
$COMPOSE exec mysql mysql -u mimeet_user -p mimeet

# 查看系統資源
df -h        # 磁碟
free -h      # 記憶體
docker stats --no-stream   # 各容器資源

# 重啟所有服務
$COMPOSE restart
```

## 12. 發版流程 SOP

```
開發完成 → PR merge to main
       ↓
GitHub Actions 自動執行：
  ① 後端單元測試
  ② 建置 Docker Image → push to GHCR
  ③ 建置前台 / 後台靜態資源
  ④ SSH 進 Droplet：
     - rsync 更新靜態資源
     - docker pull 最新 image
     - 執行 Migration
     - 滾動重啟容器
     - 清除 Laravel 快取
     - reload Nginx
       ↓
確認 Uptime Robot 無告警（5 分鐘）
       ↓
確認 https://mimeet.tw 前台正常
       ↓
完成 ✅
```

### 12.1 緊急回滾

```bash
ssh deploy@{Droplet-IP}
cd /var/www/mimeet

# 後端回滾（指定前一個 short SHA）
export IMAGE_TAG={前一個 SHA}
docker compose -f docker-compose.prod.yml up -d --no-deps app reverb horizon scheduler

# 前台靜態資源回滾
rsync -a --delete /var/www/mimeet/frontend-dist.bak/ /var/www/mimeet/frontend-dist/
sudo systemctl reload nginx

# Migration 回滾（若有）
docker compose -f docker-compose.prod.yml exec app php artisan migrate:rollback --step=1
```
