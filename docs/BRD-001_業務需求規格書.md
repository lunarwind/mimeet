## 1. 執行摘要

### 1.1 商業機會
台灣18-28歲未婚女性約39.5萬人，其中約5%（1.97萬人）對包養/一夜情服務有潛在需求。現有平台多為國際品牌，本土化不足且用戶體驗欠佳。我們的解決方案是建立**專業的包養交友平台**，配合創新的**QR碼約會驗證系統**，填補台灣本土市場空白。

### 1.2 財務預測
- **投資需求**：150萬台幣
- **收入預測**：第一年87萬，第二年193萬，第三年322萬台幣
- **投資回報**：預期3-5年戰略併購，20-50倍回報
- **市場地位**：3年內成為台灣包養交友平台前三名

### 1.3 核心競爭優勢
1. **市場定位精準**：專注包養/一夜情垂直市場
2. **技術創新**：QR碼約會驗證系統
3. **本土化優勢**：深度理解台灣用戶需求和文化
4. **成本控制**：精實營運，18個月實現盈虧平衡

## 2. 市場分析

### 2.1 目標用戶群體分析

#### 2.1.1 核心目標群體：18-28歲未婚女性
- **18-28歲女性總數**：約52.7萬人
- **年齡分層**：
  - 18-24歲：約31.6萬人（60%）
  - 24-28歲：約21.1萬人（40%）
- **未婚女性數量**：約39.5萬人（75%未婚率）

#### 2.1.2 市場定位與需求
**平台定位**：專注包養和一夜情需求的交友平台
- **用戶需求**：經濟支援、短期關係、靈活交友
- **服務特色**：匿名性、安全性、便利性

#### 2.1.3 實際市場規模分析
**TAM (Total Addressable Market)**：
- 潛在興趣用戶：約1.97萬人（5%滲透率）
- 可觸及用戶：約5,923人（30%平台知曉率）

**SAM (Serviceable Addressable Market)**：
- 實際註冊用戶：約888人（15%轉換率）
- 活躍女性用戶：約532人
- 配套男性用戶：約380人

**SOM (Serviceable Obtainable Market)**：
- 3年目標用戶：2,500人
- 付費用戶：450人
- 預期年收入：322萬台幣

#### 2.1.4 用戶需求分析

**主要痛點**：
1. **匿名性需求**（92%）：希望保護個人隱私和社會形象
2. **安全性擔憂**（87%）：擔心遇到不實用戶或危險情況
3. **金錢透明度**（78%）：希望明確的經濟條件和支付方式
4. **時間靈活性**（69%）：不希望有長期承諾和情感包袱
5. **品質篩選**（65%）：希望平台能篩選優質對象

**包養市場行情調研**：
- **月包行情**：1-5萬台幣（根據條件和頻率）
- **單次約會**：5,000-15,000台幣
- **主要支付方式**：現金、轉帳、禮物
- **平均關係維持時間**：3-8個月

### 3.3 定價策略

#### 3.3.1 定價原則
1. **價值導向定價**：基於用戶獲得的價值制定價格
2. **競爭性定價**：參考競爭對手，保持競爭力
3. **心理定價**：使用399、599等心理價格點
4. **階梯式定價**：提供多種選擇，引導用戶升級

#### 3.3.2 促銷策略
```
新用戶優惠：
├─首月半價體驗：NT$199
├─前三個月優惠：NT$299/月
└─年費新用戶折扣：直接8折

節慶促銷：
├─情人節特惠：年費7折
├─聖誕節活動：送贈點數
└─週年慶：全方案85折

會員升級：
├─升級獎勵：免費延長一個月
├─忠誠用戶：第13個月免費
└─推薦獎勵：推薦成功送30天會員
```

#### 3.3.3 一次性體驗訂閱方案（新建置優先）

```
體驗訂閱（新用戶限定，每帳號限購一次）：
├─價格：由管理員設定（預設 NT$199）
├─效期：30 天付費功能體驗
├─無自動續費功能（到期即止）
├─不可與其他折扣優惠併用
└─目的：降低首次付費門檻，提升付費轉換率
```

**體驗訂閱業務邏輯：**
- 新用戶於「會員商城」頁面優先顯示「新手體驗價」，正式方案次之
- 購買後系統記錄該帳號已使用體驗訂閱，後續商城不再顯示體驗方案
- 體驗期結束後，系統寄送 Email 引導用戶升級正式訂閱

#### 3.3.4 折扣管理機制

| 設定維度 | 說明 |
|---|---|
| 折扣對象 | 月費、季費、年費各自獨立設定 |
| 折扣形式 | 百分比（如 85 折）或固定金額折抵（如折 100 元） |
| 有效期間 | 可設定起訖日期，或永久生效 |
| 即時生效 | 後台儲存後立即反映於前台顯示價格 |
| 操作權限 | 僅具備 `settings.pricing` 權限的管理員可變更 |

#### 3.3.5 取消訂閱政策

```
取消訂閱業務規則：
├─用戶可隨時提交「取消自動續費」申請（非立即退款）
├─申請後服務持續至當期訂閱到期日
├─到期後系統自動降回「驗證會員」等級
├─同一訂閱週期內只能提交一次取消申請
└─管理員後台可查看所有待處理的取消申請並執行停止續費操作

退款政策（初版）：
└─原則上不提供退款，使用者在購買頁面需確認此政策
    （後續可依法規要求或用戶回饋調整）
```
