## UF-01 主用戶旅程總覽

> **說明：** 展示三類主要用戶角色（未登入訪客、已登入一般用戶、付費會員）的關鍵路徑分支。

```mermaid
flowchart TD
    OPEN([開啟 MiMeet]) --> AUTH{已登入？}

    AUTH -->|否| LANDING[Landing Page\n品牌首頁]
    LANDING --> CHOICE{用戶選擇}
    CHOICE -->|新用戶| REG[註冊流程\nUF-02]
    CHOICE -->|既有用戶| LOGIN[登入流程\nUF-03]

    AUTH -->|是| STATUS{帳號狀態}
    STATUS -->|停權| SUSPEND[停權頁面\n只能申訴]
    STATUS -->|正常| MEMBER_TYPE{會員等級}

    MEMBER_TYPE -->|註冊會員 Lv0| LV0_PATHS
    MEMBER_TYPE -->|驗證會員 Lv1| LV1_PATHS
    MEMBER_TYPE -->|付費會員 Lv3| LV3_PATHS

    subgraph LV0_PATHS [Level 0 — 註冊會員]
        direction TB
        L0A[瀏覽探索列表]
        L0B[基礎搜尋]
        L0C[每日 10 則訊息上限]
        L0D[引導完成驗證]
    end

    subgraph LV1_PATHS [Level 1 — 驗證會員]
        direction TB
        L1A[完整個人資料]
        L1B[每日 30 則訊息]
        L1C[進階驗證入口]
        L1D[引導升級付費]
    end

    subgraph LV3_PATHS [Level 3 — 付費會員]
        direction TB
        L3A[無限聊天 + 已讀]
        L3B[QR約會驗證]
        L3C[動態瀏覽與發布]
        L3D[誰來看我 + 收藏]
        L3E[VIP功能: 隱身/廣播]
    end

    LV0_PATHS --> UPGRADE_PROMPT[升級提示]
    LV1_PATHS --> UPGRADE_PROMPT
    UPGRADE_PROMPT --> SHOP[會員商城\nUF-07]
```

## UF-02 註冊與分層驗證流程

> **說明：** 涵蓋三步驟註冊 → Email 驗證 → 手機 SMS 驗證 → 進階驗證（依性別分流）。

```mermaid
flowchart TD
    START([訪客進入 /register]) --> STEP1

    subgraph STEP1 [Step 1：基本資料]
        S1A[選擇性別\n甜爹 / 甜心] --> S1B[填入暱稱]
        S1B --> S1C[填入生日]
        S1C --> AGE{年滿 18 歲？}
        AGE -->|否| AGE_BLOCK[顯示錯誤\n「未滿18歲無法加入」]
        AGE -->|是| S1D[下一步 →]
    end

    subgraph STEP2 [Step 2：帳號資料]
        S2A[填入 Email] --> S2B[設定密碼]
        S2B --> S2C[輸入手機號碼]
        S2C --> S2D[勾選條款 + 成年確認]
        S2D --> TERMS{兩項均勾？}
        TERMS -->|否| S2D
        TERMS -->|是| S2E[送出註冊]
    end

    subgraph STEP3 [Step 3：Email 驗證]
        S3A[系統寄送驗證信] --> S3B[用戶點擊驗證連結\n或輸入 6 位 OTP]
        S3B --> EMAIL_OK{驗證成功？}
        EMAIL_OK -->|逾時 / 失敗| S3C[重新發送\n60秒冷卻]
        S3C --> S3B
        EMAIL_OK -->|成功| S3D[✅ Email 已驗證]
    end

    subgraph SMS [SMS 手機驗證]
        SMS_A[系統發送簡訊驗證碼] --> SMS_B[用戶輸入 6 位數]
        SMS_B --> SMS_OK{驗證成功？}
        SMS_OK -->|失敗| SMS_C[重試 / 重新發送]
        SMS_C --> SMS_B
        SMS_OK -->|成功| SMS_D[✅ 手機已驗證\n升為「驗證會員 Lv1」]
    end

    STEP1 --> STEP2 --> STEP3 --> SMS --> EXPLORE

    EXPLORE[跳轉至 /app/explore\n開始使用基礎功能]

    EXPLORE --> ADV_GATE{是否進行\n進階驗證？}
    ADV_GATE -->|稍後再說| BASIC_USE[以驗證會員身份使用]
    ADV_GATE -->|是| GENDER_SPLIT{性別判斷}

    subgraph FEMALE_VERIFY [女性進階驗證]
        FV1[系統顯示隨機 6 位數驗證碼\n有效 10 分鐘] --> FV2[用戶手持驗證碼拍攝自拍]
        FV2 --> FV3[上傳照片至後台]
        FV3 --> FV4[管理員人工審核]
        FV4 --> FV5{審核結果}
        FV5 -->|通過| FV6[✅ 進階驗證完成\n誠信分數 +15]
        FV5 -->|不通過| FV7[通知用戶原因\n可重新提交]
    end

    subgraph MALE_VERIFY [男性進階驗證（信用卡）]
        MV1[跳轉綠界信用卡驗證頁面] --> MV2[輸入信用卡資訊\n收費 NT$100]
        MV2 --> MV3{付款成功？}
        MV3 -->|失敗| MV4[顯示失敗訊息\n引導重試]
        MV3 -->|成功| MV5[✅ 進階驗證完成\n誠信分數 +15]
    end

    GENDER_SPLIT -->|女性| FEMALE_VERIFY
    GENDER_SPLIT -->|男性| MALE_VERIFY

    FV6 --> LEVEL2[升為「進階驗證會員 Lv2」]
    MV5 --> LEVEL2

    style FV6 fill:#065f46,color:#fff
    style MV5 fill:#065f46,color:#fff
    style LEVEL2 fill:#1e3a5f,color:#fff
    style AGE_BLOCK fill:#7f1d1d,color:#fff
```

## UF-03 登入與帳號狀態處理

> **說明：** 登入後的狀態分叉，含停權判斷、申訴流程、忘記密碼。

```mermaid
flowchart TD
    START([進入 /login]) --> FILL[填入 Email + 密碼]
    FILL --> SUBMIT[送出登入]
    SUBMIT --> AUTH_RES{驗證結果}

    AUTH_RES -->|Email / 密碼錯誤| ERR1[Toast：帳號或密碼不正確]
    ERR1 --> FILL

    AUTH_RES -->|連續錯誤 5 次| RATE[顯示「請稍後再試」\nIP 冷卻封鎖]

    AUTH_RES -->|成功| CREDIT_CHECK{誠信分數檢查}

    CREDIT_CHECK -->|分數 ≤ 0\n自動停權| SUSPEND_FLOW
    CREDIT_CHECK -->|分數正常| ROLE_CHECK{會員等級}

    subgraph SUSPEND_FLOW [停權流程]
        SP1[跳轉 /suspended] --> SP2[顯示停權原因\n及申訴入口]
        SP2 --> SP3[填寫申訴表單\n可附圖說明]
        SP3 --> SP4[申訴送出，等待審核]
        SP4 --> SP5{管理員審核}
        SP5 -->|駁回| SP6[維持停權\nEmail 通知]
        SP5 -->|通過| SP7[補回分數至 30+ 分\n自動解除停權]
        SP7 --> RESUME[恢復正常使用]
    end

    ROLE_CHECK -->|任何等級| MAIN[進入主畫面 /app/explore]

    subgraph FORGOT [忘記密碼流程]
        FP1[點擊「忘記密碼」] --> FP2[輸入註冊 Email]
        FP2 --> FP3[系統寄送重設連結]
        FP3 --> FP4[點擊連結進入重設頁]
        FP4 --> FP5[輸入新密碼 + 確認]
        FP5 --> FP6[✅ 密碼重設成功]
        FP6 --> START
    end

    START -.->|忘記密碼| FORGOT

    style SP7 fill:#065f46,color:#fff
    style SP6 fill:#7f1d1d,color:#fff
    style SUSPEND_FLOW fill:#1c1c1c,color:#fff
```

## UF-06 QR碼約會驗證流程

> **說明：** 平台核心差異化功能。從聊天中發起邀請，到雙方掃碼完成驗證並獲得誠信分數。

```mermaid
flowchart TD
    START([用戶A 在聊天頁點擊 QR 約會 icon]) --> INVITE_FORM

    subgraph INVITE_FORM [發起邀請表單]
        IF1[填入約會日期 + 時間]
        IF2[填入地點名稱]
        IF3[確認發送邀請]
        IF1 --> IF2 --> IF3
    end

    IF3 --> SEND_INVITE[系統建立約會記錄\nPOST /api/v1/dates\n生成唯一 QR Token]

    SEND_INVITE --> USER_B[推播通知給用戶B]

    USER_B --> B_RESPONSE{用戶B 回應}
    B_RESPONSE -->|拒絕| CANCELLED[約會取消\n通知用戶A]
    B_RESPONSE -->|接受| ACCEPTED[約會狀態 → 進行中]

    ACCEPTED --> BOTH_WAIT[雙方等待約定時間]

    BOTH_WAIT --> TIME_WINDOW{進入有效掃碼時間窗\n約會時間 -30 ~ +30 分鐘}

    TIME_WINDOW -->|未到時間| COUNTDOWN[App 顯示倒數計時器]
    COUNTDOWN --> TIME_WINDOW
    TIME_WINDOW -->|超過 +30 分鐘| EXPIRED[QR Token 失效\n約會狀態 → 過期]

    TIME_WINDOW -->|進入時間窗| SCAN_READY

    subgraph SCAN_READY [雙方就位掃碼]
        SR1[各自進入 /app/dates 約會頁]
        SR1 --> SR2[點擊「立即掃碼」按鈕]
        SR2 --> SR3[開啟全螢幕相機掃碼框]
        SR3 --> SR4[掃描對方 QR Code]
    end

    SCAN_READY --> VERIFY_API[POST /api/v1/dates/:id/verify\n{ qr_token, gps_lat, gps_lng }]

    VERIFY_API --> TOKEN_CHECK{QR Token 有效？}
    TOKEN_CHECK -->|無效 / 已使用| VER_FAIL[驗證失敗\n顯示錯誤訊息]

    TOKEN_CHECK -->|有效| GPS_CHECK{用戶是否提供 GPS？}

    GPS_CHECK -->|未提供 GPS| NO_GPS[不影響驗證成功\n但獎勵分數較低]
    GPS_CHECK -->|提供 GPS| RANGE_CHECK{雙方距離\n≤ 500 公尺？}

    RANGE_CHECK -->|超出範圍| OUT_RANGE[GPS 超出範圍\n視為未提供 GPS\n仍可完成驗證]
    RANGE_CHECK -->|在範圍內| GPS_PASS[GPS 驗證通過]

    GPS_PASS --> SCORE_HIGH[雙方各獲得 +5 分\n（後台可調整）]
    NO_GPS --> SCORE_LOW[雙方各獲得 +2 分]
    OUT_RANGE --> SCORE_LOW

    SCORE_HIGH --> SUCCESS
    SCORE_LOW --> SUCCESS

    SUCCESS[✅ 約會驗證完成！\n約會狀態 → 已完成]
    SUCCESS --> NOTIFY[推播通知雙方\n「恭喜完成見面驗證，誠信分數 +N」]
    SUCCESS --> LOG[系統記錄分數變更日誌]
    SUCCESS --> PREVENT_DUPE[同一約會\n防止重複驗證]

    style SUCCESS fill:#065f46,color:#fff
    style SCORE_HIGH fill:#1a3a1a,color:#fff
    style SCORE_LOW fill:#1e3a5f,color:#fff
    style EXPIRED fill:#7f1d1d,color:#fff
    style VER_FAIL fill:#7f1d1d,color:#fff
    style CANCELLED fill:#4b5563,color:#fff
```

## UF-07 訂閱付費流程

```mermaid
    HAS_SUB -->|無| PLAN_SECTION[顯示方案選擇區]

    MY_MEMBER --> PLAN_SECTION

    subgraph TRIAL_CHECK [體驗方案判斷]
        TC1{此帳號是否\n曾購買體驗方案？}
        TC1 -->|從未購買| SHOW_TRIAL[顯示「新手體驗價」\nNT$199 / 30天 / 不自動續費]
        TC1 -->|已購買過| HIDE_TRIAL[不顯示體驗方案]
    end

    PLAN_SECTION --> TRIAL_CHECK

    subgraph PLAN_LIST [一般訂閱方案]
```
