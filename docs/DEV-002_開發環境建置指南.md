# [DEV-002] MiMeet 開發環境建置指南

**文檔版本：** v1.0  
**適用對象：** 所有開發人員（前端 / 後端 / 全端）  
**更新日期：** 2026年3月

---

## 1. 前置需求

開始前請確認已安裝以下工具：

| 工具 | 版本需求 | 說明 |
|------|----------|------|
| Docker Desktop | 4.x+ | 所有服務容器化，統一環境 |
| Docker Compose | 2.x+ | 多容器編排（通常與 Docker Desktop 同捆） |
| Node.js | 20.x LTS | 前端開發（使用 nvm 管理版本） |
| Git | 2.40+ | 版本控制 |
| VS Code | 最新版 | 建議 IDE |

> **注意：** 後端 PHP / Laravel 環境完全執行於 Docker 容器內，**不需要**在本機安裝 PHP 或 Composer。

## 4. Docker Compose 設定

### 4.1 `docker-compose.yml`（服務定義）

```yaml
version: '3.9'

services:
  # ── 後端 Laravel API ─────────────────────────
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile.dev
    container_name: mimeet_backend
    restart: unless-stopped
    volumes:
      - ./backend:/var/www/html
      - ./docker/php/php.ini:/usr/local/etc/php/conf.d/custom.ini
    environment:
      - PHP_IDE_CONFIG=serverName=mimeet
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy

  # ── Nginx ────────────────────────────────────
  nginx:
    image: nginx:1.25-alpine
    container_name: mimeet_nginx
    restart: unless-stopped
    ports:
      - "${APP_PORT:-8000}:80"
    volumes:
      - ./backend:/var/www/html
      - ./docker/nginx/default.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - backend

  # ── Laravel Reverb（WebSocket）────────────────
  reverb:
    build:
      context: ./backend
      dockerfile: Dockerfile.dev
    container_name: mimeet_reverb
    restart: unless-stopped
    command: php artisan reverb:start --host=0.0.0.0 --port=8080
    ports:
      - "8080:8080"
    volumes:
      - ./backend:/var/www/html
    depends_on:
      - backend

  # ── Queue Worker ─────────────────────────────
  queue:
    build:
      context: ./backend
      dockerfile: Dockerfile.dev
    container_name: mimeet_queue
    restart: unless-stopped
    command: php artisan horizon
    volumes:
      - ./backend:/var/www/html
    depends_on:
      - backend

  # ── Scheduler ────────────────────────────────
  scheduler:
    build:
      context: ./backend
      dockerfile: Dockerfile.dev
    container_name: mimeet_scheduler
    restart: unless-stopped
    command: sh -c "while true; do php artisan schedule:run; sleep 60; done"
    volumes:
      - ./backend:/var/www/html
    depends_on:
      - backend

  # ── MySQL 8.0 ────────────────────────────────
  mysql:
    image: mysql:8.0
    container_name: mimeet_mysql
    restart: unless-stopped
    ports:
      - "3306:3306"
    environment:
      MYSQL_DATABASE: ${DB_DATABASE}
      MYSQL_USER: ${DB_USERNAME}
      MYSQL_PASSWORD: ${DB_PASSWORD}
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD}
    volumes:
      - mysql_data:/var/lib/mysql
      - ./docker/mysql/my.cnf:/etc/mysql/conf.d/custom.cnf
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

  # ── Redis 7.0 ────────────────────────────────
  redis:
    image: redis:7.0-alpine
    container_name: mimeet_redis
    restart: unless-stopped
    ports:
      - "6379:6379"
    command: redis-server --requirepass ${REDIS_PASSWORD}
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "--no-auth-warning", "-a", "${REDIS_PASSWORD}", "ping"]
      interval: 10s
      timeout: 5s
      retries: 5

  # ── MinIO（本地 S3 模擬）───────────────────────
  minio:
    image: minio/minio:latest
    container_name: mimeet_minio
    ports:
      - "9000:9000"
      - "9001:9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    command: server /data --console-address ":9001"
    volumes:
      - minio_data:/data

  # ── Mailpit（Email 攔截，取代 Mailhog）────────
  mailpit:
    image: axllent/mailpit:latest
    container_name: mimeet_mailpit
    ports:
      - "${MAILPIT_PORT:-8025}:8025"  # Web UI
      - "1025:1025"                    # SMTP

  # ── phpMyAdmin（可選，資料庫管理介面）─────────
  phpmyadmin:
    image: phpmyadmin:latest
    container_name: mimeet_phpmyadmin
    ports:
      - "${PHPMYADMIN_PORT:-8080}:80"
    environment:
      PMA_HOST: mysql
      PMA_USER: root
      PMA_PASSWORD: ${DB_ROOT_PASSWORD}

volumes:
  mysql_data:
  redis_data:
  minio_data:
```

## 5. 後端初始化步驟

```bash
# Step 1：啟動所有容器
docker compose up -d

# Step 2：確認容器狀態
docker compose ps

# Step 3：安裝 Composer 套件
docker compose exec backend composer install

# Step 4：產生 App Key
docker compose exec backend php artisan key:generate

# Step 5：執行資料庫遷移
docker compose exec backend php artisan migrate

# Step 6：填入基礎資料（Seeder）
docker compose exec backend php artisan db:seed

# Step 7：建立儲存連結（本地圖片服務用）
docker compose exec backend php artisan storage:link

# Step 8：建立 MinIO Bucket
docker compose exec backend php artisan s3:create-bucket
```

### 5.1 初始 Seeder 內容

執行 `db:seed` 會建立：

| 資料 | 說明 |
|------|------|
| `AdminUserSeeder` | 建立預設 super_admin（帳號/密碼於 `.env`） |
| `AdminRoleSeeder` | 建立三個角色：super_admin / admin / cs |
| `AdminPermissionSeeder` | 建立所有權限節點 |
| `SubscriptionPlanSeeder` | 建立月費/季費/年費方案 |
| `TrialPlanSeeder` | 建立體驗價方案（預設 NT$199） |
| `TaiwanLocationSeeder` | 台灣縣市/地區資料 |

## 10. 常用指令速查

### 後端（透過 Docker）

```bash
# 執行 Artisan 指令
docker compose exec backend php artisan <command>

# 進入後端容器 Shell
docker compose exec backend bash

# 執行單一 Migration
docker compose exec backend php artisan migrate --path=database/migrations/xxxx.php

# 重置資料庫（開發用）
docker compose exec backend php artisan migrate:fresh --seed

# 清除快取
docker compose exec backend php artisan optimize:clear

# 查看路由列表
docker compose exec backend php artisan route:list --path=api/v1

# 執行測試
docker compose exec backend php artisan test

# 執行特定測試
docker compose exec backend php artisan test --filter=UserRegistrationTest
```

### 前端

```bash
# 開發伺服器
npm run dev

# 建置生產版本
npm run build

# 型別檢查
npm run type-check

# ESLint 檢查
npm run lint

# 執行測試（Vitest）
npm run test
```

### Docker 管理

```bash
# 啟動所有服務
docker compose up -d

# 停止所有服務
docker compose down

# 重啟特定服務
docker compose restart backend

# 查看 Log
docker compose logs -f backend
docker compose logs -f queue

# 刪除所有 Volume（完全重置）
docker compose down -v
```

## 11. 疑難排解

### 問題：MySQL 連線失敗
```bash
# 確認 MySQL 容器健康狀態
docker compose ps mysql
# 應顯示 "healthy"

# 查看 MySQL Log
docker compose logs mysql
```

### 問題：Composer 安裝超時
```bash
# 進入容器後設定鏡像源
docker compose exec backend composer config -g repo.packagist composer https://mirrors.aliyun.com/composer/
docker compose exec backend composer install
```

### 問題：Storage 權限錯誤
```bash
docker compose exec backend chmod -R 775 storage bootstrap/cache
docker compose exec backend chown -R www-data:www-data storage bootstrap/cache
```

### 問題：前台 API 請求 CORS 錯誤
確認 `backend/.env` 中 `SANCTUM_STATEFUL_DOMAINS` 包含前台本地 URL：
```dotenv
SANCTUM_STATEFUL_DOMAINS=localhost:3000,127.0.0.1:3000
```
