### Issue G-001 — nginx 缺少 HSTS + CSP（VULN-004 部分修復）

**嚴重性：** 🟠 High  
**規格位置：** DEV-012 §8（VULN-004）  
**受影響檔案：** `docker/nginx/default.conf`

**現狀（已有）：**
```nginx
add_header X-Frame-Options "DENY";
add_header X-Content-Type-Options "nosniff";
add_header Referrer-Policy "strict-origin-when-cross-origin";
add_header Permissions-Policy "geolocation=(), microphone=(), camera=()";
```

**缺失：**
```nginx
# 1. HSTS — 強制 HTTPS，防 SSL stripping
add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

# 2. CSP — 防 XSS inline script injection
add_header Content-Security-Policy "default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; connect-src 'self' https://api.mimeet.online wss://api.mimeet.online;" always;

# 3. 隱藏 server banner
server_tokens off;
```

**影響：**
- 缺少 HSTS → 用戶首次 HTTP 訪問不被強制升級至 HTTPS，易遭 SSL Stripping 降級攻擊
- 缺少 CSP → 若存在 XSS 漏洞，攻擊者可注入任意外部腳本
- `server_tokens on`（預設）→ nginx 版本號在 error page / response header 中洩露，方便攻擊者針對版本漏洞

### Issue G-003 — `user_photos` 架構偏差（改用 `avatar_slots` JSON）

**嚴重性：** 🟡 Medium  
**規格位置：** DEV-006 §3.1（用戶模組）  
**受影響檔案：** `backend/app/Models/User.php`、無 `user_photos` migration

**DEV-006 規格定義：**
```
user_photos: id, user_id, url, slot_index, is_primary, created_at
```

**實際實作：**
```php
// users 表的 JSON 欄位取代獨立的 user_photos 表
protected $casts = [
    'avatar_slots' => 'array',   // JSON 欄位直接儲存多張照片
];
```

**差異分析：**

| 能力 | DEV-006 `user_photos` 表 | 實際 `avatar_slots` JSON |
|------|--------------------------|--------------------------|
| 每張照片獨立軟刪除 | ✅ | ❌ |
| 照片 URL 外部查詢（JOIN） | ✅ | ❌ |
| 上傳時間 per-photo | ✅ | ❌ |
| 查詢效能（單表 scan） | 需索引 | JSON 欄位直接讀 |

**影響：** 規格書若被前端開發者照單全收，期望 `/user_photos` 路由或獨立照片管理端點，會找不到對應後端。需更新 DEV-006 §3.1 反映實際設計。

**規格位置：** DEV-012 §11（VULN-008）

**影響：** 後台 admin Sanctum token 被竊取後，攻擊者可從任意 IP 使用該 token 存取所有後台功能（結合 Issue F-001 RBAC 未掛載，攻擊面更大）。

### Issue G-007 — CORS localhost 來源未做環境隔離（VULN-011 部分修復）

**嚴重性：** 🔵 Low  
**規格位置：** DEV-012 §14（VULN-011）  
**受影響檔案：** `backend/config/cors.php:15–21`

**現狀：**
```php
'allowed_origins' => [
    'http://localhost:5173',   // Vite 前台 dev server
    'http://localhost:5174',   // Vite 後台 dev server
    'https://mimeet.online',
    'https://www.mimeet.online',
    'https://admin.mimeet.online',
],
```

**問題：** 生產環境 config 包含 localhost dev 來源。若同主機上有其他服務在 `:5173/:5174` 監聽（或未來有容器端口衝突），瀏覽器 CORS 政策會允許該服務讀取 API 回應。

**建議：** 透過 `.env` 分離：
```php
'allowed_origins' => array_filter(explode(',', env('CORS_ALLOWED_ORIGINS', 'https://mimeet.online,https://www.mimeet.online,https://admin.mimeet.online'))),
```
生產 `.env` 不含 localhost；本機 `.env` 加入 localhost。
