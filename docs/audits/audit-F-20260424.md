## Issue #F-001

**Pass：** P3（回應結構）、P6（認證）  
**規格位置：** docs/API-002 §1.2  
**規格內容：**
```
Token 由後台登入 API 取得，有效期 8 小時（與前台用戶 24h 不同，安全性更高）。
```
**程式碼位置：** backend/app/Http/Controllers/Api/V1/AdminController.php:44  
**程式碼現況：**
```php
$token = $admin->createToken("admin-token-{$loginIp}", ['admin'])->plainTextToken;
// 無獨立 expiration 設定，沿用全域 SANCTUM_TOKEN_EXPIRATION=1440（24h）
```
**差異說明：** 規格明確要求 admin token 8小時有效（480分鐘），理由是安全性更高。但程式碼使用全域 `SANCTUM_TOKEN_EXPIRATION=1440`，admin token 實際有效期 24h。若後台帳號遭竊，攻擊者有兩倍時間可利用 token。Laravel Sanctum 支援以 abilities/scope 設定不同 TTL。  
**等級：** 🟠 High  
**建議方案：**
- Option A（推薦）：在 `createToken()` 後立即更新 token 的 expires_at：`PersonalAccessToken::where('id', $tokenId)->update(['expires_at' => now()->addMinutes(480)])`
- Option B：新增 `ADMIN_TOKEN_EXPIRATION=480` 環境變數並在 AdminController 套用
**推薦：** A（不影響前台用戶 token TTL）

## Issue #F-002

**Pass：** P1（路由層）  
**規格位置：** docs/API-002 §4 速查表  
**規格內容：** `GET /members/{id}/credit-logs` — members.view 權限  
**程式碼位置：** backend/routes/api.php（搜尋無此路由）  
**程式碼現況：** memberDetail() 在回應中包含當前 credit_score，但無獨立的 credit history 端點。  
**差異說明：** 後台管理員無法查詢特定會員的完整誠信分數異動記錄。這是後台監控用戶行為的重要功能（識別惡意行為、驗證加減分是否正常）。`credit_score_histories` 表已存在數據。  
**等級：** 🟠 High  
**建議方案：**
- Option A（推薦）：新增路由 + AdminController 方法，查詢 `CreditScoreHistory::where('user_id', $id)->paginate()`
**推薦：** A

## Issue #F-010

**Pass：** P3（回應結構）  
**規格位置：** docs/API-002 §1.2（token 8小時）  
**程式碼位置：** backend/app/Http/Controllers/Api/V1/AdminController.php:44-63  
**程式碼現況：** admin login 回應無 `expires_at` 欄位，前端無法顯示 token 剩餘時效。  
**差異說明：** 規格明確說明 8h 有效期，但登入回應不含 `expires_at`，前端 React Admin 無法顯示「您的 session 將於 X 分鐘後到期」提示。  
**等級：** 🔵 Low  
**建議方案：**
- Option A（推薦）：在登入回應加入 `expires_at: now()->addMinutes(480)->toISOString()`（配合 F-001 修復一起做）
**推薦：** A
