## Issue #G-001

**Pass：** P10（安全漏洞）  
**參考：** Audit A Issue #A-001  
**規格位置：** docs/API-001 §2.2.2.1（Authorization: Bearer 標示）  
**程式碼位置：** backend/routes/api.php:69-72  
**程式碼現況：**
```php
Route::prefix('auth')->middleware('throttle:otp')->group(function () {
    Route::post('/verify-phone/send', [...]);   // 無 auth:sanctum
    Route::post('/verify-phone/confirm', [...]);
});
```
**差異說明：** Audit A 首次記錄，本輪 Remote 確認仍未修復。無 token 可直接觸發 OTP SMS，造成 SMS 費用濫用風險。  
**等級：** 🔴 Critical  
**修法：** 加入 `auth:sanctum` 至此路由群組

## Issue #G-002

**Pass：** P10（安全漏洞）  
**規格位置：** DEV-012（若存在）  
**程式碼位置：** backend/config/reverb.php  
**程式碼現況：**
```php
'allowed_origins' => ['*'],   // WebSocket handshake 不限制 origin
```
**差異說明：** Reverb WebSocket 連線端點允許任何 origin 發起連線。雖然 private channel 訂閱仍需 Bearer token（channels.php），但 `*` 的設定：(1) 允許跨站 WebSocket 連線（CSRF-like 攻擊向量）；(2) 任何網站可嘗試 WebSocket handshake 並消耗伺服器資源。CORS 主設定（`config/cors.php`）已限定特定 origin，Reverb 應與其一致。  
**等級：** 🟠 High  
**建議方案：**
- Option A（推薦）：改為 `['https://mimeet.online', 'https://admin.mimeet.online']`
**推薦：** A

## Issue #G-008

**Pass：** P3（索引比對）  
**規格位置：** DEV-006 §7（查詢索引：WHERE 條件常用欄位必須加索引）  
**程式碼現況（Remote 確認）：** `SHOW INDEX FROM messages WHERE Index_type='FULLTEXT'` → 無結果。ChatService::searchMessages() 使用 `content LIKE '%keyword%'`，無 FULLTEXT 支援。  
**差異說明：** searchMessages（F20 功能）對 messages.content 做 LIKE 模糊查詢，不走任何索引，隨訊息量成長將退化為全表掃描。DEV-006 §7 明定「查詢索引：WHERE 條件常用欄位必須加索引」。  
**等級：** 🟡 Medium  
**建議方案：**
- Option A（推薦）：加 migration：`$table->fullText('content')` 並改用 `MATCH(content) AGAINST(?)` 全文搜尋
- Option B：加普通 index（`$table->index('content')`），但對前綴 LIKE 效果有限
**推薦：** A
