## 2. 🔴 Critical Issues 全清單

### CR-1：`useImageUpload.ts` 呼叫不存在的路由（D-001）
**報告：** Audit D  
**規格位置：** API-001 §16.1（承認無統一端點）  
**程式碼位置：** `frontend/src/composables/useImageUpload.ts:85`  
**摘要：** 前端 `useImageUpload` composable 對所有上傳情境（頭像、個人照片、舉報截圖）一律呼叫 `POST /uploads`，但後端 `routes/api.php` 不存在此路由。`AccountView.vue:317`、`ReportsView.vue:12` 使用此 composable，導致所有照片上傳均 404 失敗。  
**建議工時：** S（補 `/uploads` 路由統一接收，轉發至現有上傳邏輯）

---

### CR-2：後台 RBAC Middleware 從未掛載（F-001）
**報告：** Audit F  
**規格位置：** API-002 §10（RBAC 角色權限）  
**程式碼位置：** `backend/app/Http/Kernel.php:67`（registered）；`backend/routes/api.php`（未使用）  
**摘要：** `CheckAdminPermission` middleware 已在 `Kernel.php:67` 以 `admin.permission` 別名登錄，但掃描全部 admin 路由後確認：任何路由均未掛載 `->middleware('admin.permission', ...)`。所有已登入管理員不論角色（viewer/operator/super_admin）均可存取全部後台功能，RBAC 系統完全失效。  
**建議工時：** M（在 `routes/api.php` 各 admin 路由群組補上 middleware 呼叫）

---

### CR-3：舉報 `type` 前端送 number，後端驗 string enum（H-001）
**報告：** Audit H  
**規格位置：** API-001 §8.1  
**程式碼位置：** `frontend/src/api/reports.ts:8`（type: number）；`backend/app/Http/Controllers/Api/V1/ReportController.php:24`（in:fake_photo,...)  
**摘要：** 前端 `CreateReportPayload.type` 定義為 `number`，送出數字 1–5。後端 validation rule：`required|in:fake_photo,harassment,spam,scam,inappropriate,system_issue,other`。數字型別永遠不在字串 enum 內，所有 `POST /reports` 一律回傳 422 Unprocessable Entity，舉報功能完全失效。  
**建議工時：** S（後端接受數字 OR 前端改送 string，兩端對齊即可）
