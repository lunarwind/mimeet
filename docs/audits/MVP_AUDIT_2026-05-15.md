**真正問題（dead code）**：AccountView.vue 內有一段 orphan script — `photos` ref、`photoInput` ref、`triggerPhotoUpload`、`handlePhotoChange`、`removePhoto` — **`<template>` 從未引用**，使用者在 UI 上點不到。這段 dead code 走 `uploadPhoto` (內部 `/uploads` `context=profile_photo`) 確實不寫 `avatar_slots`，但**使用者實際不會 trigger**，因此沒有「上傳後消失」症狀。

**處理**：commit `<TBD>` 移除 AccountView.vue 30 行 orphan 程式碼（5 處 orphan refs/functions + 1 處 unused destructure：`uploadPhoto / uploadError / _uploadAvatar`）。已驗證 frontend build pass、無 template 引用消失。

**根因**：API-001 §3.3「頭像槽位系統」與 PRD §3 line 230「大頭貼 1 張 + 附加照片最多 2 張」表述差異，導致前端歷史開發時試圖做「頭像 vs 相冊」分離 UI，但 DB schema 只有單一 `avatar_slots` JSON 槽位（max 3），最後 template 沒接通就遺留下來。
