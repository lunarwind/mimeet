## Issue #D-001

**Pass：** P1（路由層）、P6（認證）  
**規格位置：** docs/API-001 §7.1.1  
**規格內容：**
```http
GET /api/v1/subscription/plans
# 無 Authorization: Bearer 標示 → 公開端點
```
**程式碼位置：** backend/routes/api.php:106-113  
**程式碼現況：**
```php
Route::prefix('subscriptions')->middleware('auth:sanctum')->group(function () {
    Route::get('/plans', [SubscriptionController::class, 'plans']);
    // plans 被包在 auth:sanctum 群組內
```
**差異說明：** `/subscriptions/plans` 放在 `auth:sanctum` 中介層群組內，未登入用戶請求時返回 401。規格 §7.1.1 的端點無 Authorization header 標示，表示應為公開端點。電商邏輯上用戶在決定是否購買前應能看到方案列表，不應要求先登入。Remote 確認：無 token 請求 → 401。前端 usePayment.ts 在已登入狀態下呼叫，暫無 UX 問題，但未登入狀態下（如新用戶考察頁面）無法取得方案。  
**等級：** 🟠 High  
**建議方案：**
- Option A（推薦）：將 `GET /plans` 移至 auth:sanctum 群組外（可保持其餘 subscriptions 路由在群組內）
- Option B：更新規格 §7.1.1 標注需要 Authorization（但電商體驗建議不這樣做）
**推薦：** A

## Issue #D-003

**Pass：** P2（Payload）、P3（回應結構）  
**規格位置：** docs/API-001 §10.3（行 ~2823）  
**規格內容：**
```
HTTP 201
{ "data": { "ticket_number": "R2024122000042",
    "subscription_expires_at": "2025-01-19T23:59:59Z",
    "notice": "服務將持續至訂閱到期日，到期後不自動費" } }
```
**程式碼位置：** backend/app/Http/Controllers/Api/V1/SubscriptionController.php:cancelRequest()  
**程式碼現況：**
```php
return response()->json([
    'success' => true,
    'code' => 200,         // 規格要 201
    'message' => '取消訂閱申請已提交',
    // 無 ticket_number, subscription_expires_at, notice

```
**差異說明：** 三個問題：(1) HTTP 200 vs 規格 201；(2) 缺少 `ticket_number`（用戶需記錄申請序號）；(3) 缺少 `subscription_expires_at`（告知服務何時到期）；(4) 缺少 `notice`；(5) request body 格式：規格用 `data.reason` (nested)，程式碼讀頂層 `reason`。  
**等級：** 🟡 Medium  
**建議方案：**
- Option A（推薦）：cancelRequest() 補充 `subscription_expires_at`（查 active subscription）和 `notice`；HTTP 改 201；補充 `ticket_number`（若需申請系統才需要）
**推薦：** A（subscription_expires_at 和 notice 對用戶有實際幫助，一次補齊）
