## 2. 🔴 Critical Issues（必修，影響安全或核心功能）

### A-001 / G-001 — `verify-phone/send` & `/confirm` 無 auth:sanctum

**規格：** API-001 §2.2.2.1（Authorization: Bearer 標示）  
**程式碼：** `backend/routes/api.php:69-72`（只有 throttle:otp，無 auth:sanctum）  
**影響：** 任何人不需登入即可觸發 OTP SMS → SMS 費用濫用、電話轟炸  
**Remote：** 確認仍存在（無 token POST → 200）  
**建議工時：** 0.5h  
**修法：**
```php
Route::prefix('auth')->middleware(['auth:sanctum', 'throttle:otp'])->group(function () {
    Route::post('/verify-phone/send', ...);
    Route::post('/verify-phone/confirm', ...);
});
```

---

### G-002 — Reverb WebSocket `allowed_origins: ['*']`

**規格：** DEV-012（CORS 政策）  
**程式碼：** `backend/config/reverb.php`（`'allowed_origins' => ['*']`）  
**影響：** 任何外部 origin 可建立 WebSocket 連線（channel 訂閱仍需 token，但增加跨站攻擊面）  
**建議工時：** 0.25h  
**修法：** 改為 `['https://mimeet.online', 'https://admin.mimeet.online']`

---

### H-001 — FcmService Production 返回 false（Push 通知無效）

**規格：** API-001 §10.7（通知依賴 FCM 推播）  
**程式碼：** `backend/app/Services/FcmService.php`（production 分支 return false，FCM API 被 comment 掉）  
**影響：** 全站 push 通知（新訊息、約會邀請等）在 production 完全不發出，用戶手機收不到推播  
**Remote .env：** `FCM_SERVER_KEY=`（空值）  
**建議工時：** 4-8h（含取得 Firebase Service Account）  
**修法：** 實作 FCM HTTP v1 API 或使用 `kreait/laravel-firebase` 套件
