## Phase 規劃建議

| 功能 | 類別 | 優先理由 |
|------|------|---------|
| F01–F04（註冊 / 登入 / 忘記密碼） | 用戶系統 | 最基本門檻，無此無法使用 |

## 各主要決策的 Pros & Cons

### 🟢 QR 碼約會驗證 → 放 MVP

| | 說明 |
|---|------|
| **Pro** | 最大差異化賣點，是整個平台的「身份證」；若 Phase 2 才做，用戶沒有理由在 Phase 1 留下來 |
| **Pro** | 驗證機制直接帶動誠信分數，早期沒有這個，分數系統就失去核心意義 |
| **Con** | 技術複雜（GPS + 加密 QR + 時間窗），開發成本高 |
| **Con** | GPS 在 WebView 環境需要用戶授權，可能產生 UX 摩擦 |
| **建議** | 做 MVP，但先以「時間窗驗證」為主，GPS 做選配，後台可調分值 → 降低開發風險 |

---

### 🟡 動態發布系統 → 放 Phase 2

| | 說明 |
|---|------|
| **Pro** | 增加用戶黏著度與平台活躍感；讓用戶有「除了搜尋之外」的理由來訪 |
| **Pro** | 收藏對象動態可驅動 retention |
| **Con** | 需要內容審核（AI + 人工），MVP 期間運營人力有限 |
| **Con** | 用戶量少時動態牆空洞，反而顯得冷清 |
| **Con** | 開發量大，含圖片上傳、審核 pipeline、留言系統 |
| **建議** | Phase 2 推出，同時搭配 AI 自動過濾，人工審核作輔助 |

---

### 🟡 匿名聊天室 → 放 Phase 2

| | 說明 |
|---|------|
| **Pro** | 降低初識門檻，吸引女性用戶嘗試；男性付費才能進，有驅動付費轉換的效果 |
| **Con** | 匿名機制實作複雜（同頻道固定代號 + 跨頻道不同）；管理員查證機制需要完整後台支援 |
| **Con** | 匿名環境容易產生不當言論，MVP 期間沒有足夠審核能量 |
| **建議** | PRD 已明確列為 Phase 2，維持此決策 |

---

### 🟢 新手體驗價（Trial Plan）→ 放 MVP

| | 說明 |
|---|------|
| **Pro** | 大幅降低首次付費心理門檻，是獲得第一批付費用戶的關鍵工具 |
| **Pro** | 邏輯簡單（帳號限一次 + 無自動續費），開發複雜度低 |
| **Con** | 若體驗期結束沒有做好 Email 引導，轉換率仍低 |
| **建議** | 放 MVP，搭配到期 Email 引導升級，成本低效益高 |

---

### 🔴 智能配對算法 → 放 Phase 2

| | 說明 |
|---|------|
| **Pro** | 提升配對品質感，讓高付費用戶有明顯更好的體驗 |
| **Con** | 早期用戶少（< 1,000 人），算法效果有限，複雜算法反而過度設計 |
| **Con** | 需要行為數據累積（互動率、回應率）才能調校權重 |
| **建議** | Phase 1 先用「誠信分數 + 最後上線時間」排序，夠用且好維護 |

---

### 🟡 後台統計儀表板 → 放 Phase 2（基礎版 MVP 即可）

| | 說明 |
|---|------|
| **Pro** | 管理員需要知道平台狀況，做決策依據 |
| **Con** | 圖表化開發成本高；初期可用 Google Analytics + 簡單 SQL 報表取代 |
| **建議** | MVP 只做文字型關鍵數字（新增 / 活躍 / 付費人數），圖表版放 Phase 2 |
